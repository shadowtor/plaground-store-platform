**A. KPI API (apps/api/src/routes/v1/admin/kpis.ts)**

`GET /api/v1/admin/kpis` — requires `admin.kpis.read`. No query params. Returns:
```typescript
{
  revenue: {
    todayCents: number,
    last7DaysCents: number,
    last30DaysCents: number,
    allTimeCents: number,
  },
  orders: {
    total: number, pending: number, paid: number, preparing: number, completed: number
  },
  quotes: {
    total: number, instantEstimate: number, manualReview: number, pendingAdminReview: number
  },
  queueDepths: {
    ordersNeedingAttention: number,  // PAID orders not yet moved to PREPARING
    quotesNeedingReview: number,     // MANUAL_REVIEW quotes not yet reviewed by admin
    expiringAuths: number,           // Payment.status = AUTHORIZED and authorizedAt < NOW() - 3 days
  }
}
```

Use Prisma aggregations: `prisma.order.aggregate({ _sum: { totalCents: true }, where: { status: 'PAID', createdAt: { gte: startOfToday } } })`.

`GET /api/v1/admin/kpis/auth-expiring` — returns orders with Stripe auth windows expiring within
72 hours (for the admin queue warning pattern). Returns:
```typescript
Array<{ orderId, customerEmail, totalCents, authorizedAt, expiresAt, hoursRemaining }>
```

**B. Orders API (apps/api/src/routes/v1/admin/orders.ts)**

`GET /api/v1/admin/queues/orders` — requires `admin.orders.read`. Query: `?status&cursor&limit=50`.
Returns paginated orders list for admin triage queue. Each item includes:
`{ id, status, customerEmail, totalCents, createdAt, itemCount, paymentStatus, authExpiresAt }`.
`cache: 'no-store'` is enforced by client — server uses `revalidate: 0` hint in response header.

`GET /api/v1/admin/orders/:id` — full order detail including `OrderItem[]`, `OrderEvent[]`,
`Payment` status, customer info.

`POST /api/v1/admin/orders/:id/status` — requires `admin.orders.write`. Body:
`{ status: OrderStatus, note?: string }`.
Validation: only forward transitions (no moving COMPLETED back to PAID, etc.).
Valid forward transitions:
- `PAID` → `PREPARING`
- `PREPARING` → `SHIPPED`
- `SHIPPED` → `COMPLETED`
- Any → `CANCELLED` (with required `note`)

Logic:
1. Update `Order.status`.
2. Create `OrderEvent`: `{ orderId, type: newStatus, note: note ?? 'Status updated by admin', actorUserId: req.sessionData.userId }`.
3. Call `auditLog({ actionKey: 'order.status.updated', actorUserId, targetType: 'Order', targetId: orderId, before: { status: prevStatus }, after: { status: newStatus } })`.
4. Enqueue notification: `notifyQueue.add('order-status-change', { orderId, newStatus, customerId: order.userId })`.
5. Return updated order.

**C. Quotes API (apps/api/src/routes/v1/admin/quotes.ts)**

`GET /api/v1/admin/queues/quotes` — requires `admin.quotes.read`. Query: `?status&cursor&limit=50`.
Returns quotes needing admin review (status `MANUAL_REVIEW`). Each item:
`{ id, status, uploadId, filename, format, volumeMm3, createdAt, customerId, customerEmail }`.

`GET /api/v1/admin/quotes/:id` — full quote detail including `ModelFile` info, customer info,
`QuoteReview` history if any.

`POST /api/v1/admin/quotes/:id/review` — requires `admin.quotes.write`. Body:
```typescript
{
  decision: 'APPROVED' | 'REJECTED' | 'REQUEST_CHANGES',
  note?: string,
  // If APPROVED: must include pricing override
  approvedTotalCents?: number,
  approvedLineItems?: Array<{ label: string, cents: number }>
}
```

Logic:
1. Validate quote status is `MANUAL_REVIEW` (cannot re-review an already-decided quote without
   explicit reversal, which is out of scope for Phase 1).
2. Create `QuoteReview` record: `{ quoteId, decision, note, reviewedByUserId, reviewedAt }`.
3. Update `Quote.status`: `APPROVED` → `'APPROVED'`, `REJECTED` → `'REJECTED'`,
   `REQUEST_CHANGES` → `'CHANGES_REQUESTED'`.
4. If `APPROVED`: set `Quote.totalCents = approvedTotalCents`, `Quote.lineItems = approvedLineItems`,
   `Quote.estimateType = 'ADMIN_APPROVED'`.
5. Call `auditLog({ actionKey: \`quote.review.\${decision.toLowerCase()}\`, actorUserId, targetType: 'Quote', targetId: quoteId, before: { status: 'MANUAL_REVIEW' }, after: { status: decision, note } })`.
6. Enqueue: `notifyQueue.add('quote-decision', { quoteId, decision, customerId: quote.userId })`.
7. Return updated quote.

**D. Audit log API (apps/api/src/routes/v1/admin/audit.ts)**

`GET /api/v1/admin/audit-logs` — requires `admin.audit.read`. Cursor pagination. Query params:
`?actorUserId&actionKey&targetType&targetId&from&to&cursor&limit=50`.

Returns: `{ entries: AuditLogEntry[], nextCursor: string | null }`.
Each entry: `{ id, actorUserId, actorEmail, actionKey, targetType, targetId, before, after, ipAddress, correlationId, createdAt }`.

Use cursor-based pagination on `createdAt` DESC. NEVER allow ORDER BY on the raw timestamp alone
without a tie-breaker index — use `(createdAt DESC, id DESC)` composite.
