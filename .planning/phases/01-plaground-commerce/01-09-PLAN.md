each execution context:
```
1. API Request → tenantId extracted from JWT payload by auth plugin → stored in

   every Prisma query inside the request sees the tenantId automatically.

2. Background Job → API enqueues job with job.data.tenantId explicitly set →
   worker processor wraps job body with tenantStorage.run(job.data.tenantId, async () => {...}) →
```

**C. Tenant context Fastify plugin (apps/api/src/plugins/tenant-context.ts)**

A Fastify plugin that runs as a `preHandler` on all API routes (registered globally, before route
handlers). Extracts `tenantId` from `request.sessionData.tenantId` (set by the auth plugin) and
wraps the remaining request lifecycle in `tenantStorage.run(tenantId, done)`.

```typescript
import fp from 'fastify-plugin'
import { tenantStorage } from '../lib/prisma-rls'

export default fp(async (fastify) => {
  fastify.addHook('preHandler', async (request, reply) => {
    // Public routes (e.g., webhook endpoints, health check) set their own tenant context
    // explicitly — skip if tenantId is not on the session
    if (!request.sessionData?.tenantId) return

    return new Promise<void>((resolve) => {
      tenantStorage.run(request.sessionData.tenantId, () => resolve())
    })
  })
})
```

from the related record (Order or Payment) BEFORE any tenant-scoped query, then wrap:

```typescript
  // all subsequent DB operations use tenant context
})
```

**B. Session rotation after MFA completion (apps/api/src/services/auth/session-rotation.ts)**

Create a session rotation utility:

```typescript
// apps/api/src/services/auth/session-rotation.ts
import { createSession, deleteSession, getSessionData } from '../session'

/**
 * Rotate session token after a security-sensitive step (e.g., MFA completion).
 * Invalidates the old session token and issues a new one with the same session data.
 * Prevents session fixation attacks where an attacker who captured the pre-MFA session
 * cookie could use it after MFA completion.
 */
export async function rotateSession(params: {
  oldToken: string,
  userId: string,
  tenantId: string,
  mfaVerified: true,
  role: string,
}): Promise<{ newToken: string }> {
  // 1. Read existing session data to preserve non-sensitive fields
  const existingData = await getSessionData(params.oldToken)

  // 2. Delete old session from Redis
  await deleteSession(params.oldToken)

  // 3. Create new session with mfaVerified: true
  const newToken = await createSession({
    userId: params.userId,
    tenantId: params.tenantId,
    role: params.role,
    mfaVerified: true,
    lastActiveAt: new Date().toISOString(),
    // preserve IP, user agent from existing data if available
    ipAddress: existingData?.ipAddress,
  })

  return { newToken }
}
```
