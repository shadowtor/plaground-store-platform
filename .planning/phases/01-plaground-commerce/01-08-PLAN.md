**A. Email provider interface (apps/worker/src/lib/email/index.ts)**

```typescript
// EmailMessage shape
export interface EmailMessage {
  to: string           // recipient email address
  subject: string
  html: string         // HTML body
  text: string         // plain text fallback
}

// Provider contract
export interface EmailProvider {
  sendMail(message: EmailMessage): Promise<void>
}

// Factory: select provider from EMAIL_PROVIDER env var
export function createEmailProvider(): EmailProvider {
  const provider = process.env.EMAIL_PROVIDER
  if (provider === 'microsoft-graph') return new MicrosoftGraphAdapter()
  if (provider === 'gmail') return new GmailAdapter()
  throw new Error(
    `EMAIL_PROVIDER must be 'microsoft-graph' or 'gmail'. Self-hosted SMTP and relay services are not permitted.`
  )
}
```

**B. Microsoft Graph API adapter (apps/worker/src/lib/email/graph-adapter.ts)**

Uses `@microsoft/microsoft-graph-client` with `TokenCredentialAuthenticationProvider` from
`@microsoft/microsoft-graph-client/authProviders/azureTokenCredentials`.
For OAuth2 client credentials: use `ClientSecretCredential` from `@azure/identity`.

```typescript
import { Client } from '@microsoft/microsoft-graph-client'
import { TokenCredentialAuthenticationProvider } from '@microsoft/microsoft-graph-client/authProviders/azureTokenCredentials'
import { ClientSecretCredential } from '@azure/identity'

export class MicrosoftGraphAdapter implements EmailProvider {
  private client: Client

  constructor() {
    const credential = new ClientSecretCredential(
      process.env.MS_GRAPH_TENANT_ID!,
      process.env.MS_GRAPH_CLIENT_ID!,
      process.env.MS_GRAPH_CLIENT_SECRET!
    )
    const authProvider = new TokenCredentialAuthenticationProvider(credential, {
      scopes: ['https://graph.microsoft.com/.default']
    })
    this.client = Client.initWithMiddleware({ authProvider })
  }

  async sendMail(message: EmailMessage): Promise<void> {
    await this.client.api(`/users/${process.env.MS_GRAPH_SENDER_EMAIL}/sendMail`).post({
      message: {
        subject: message.subject,
        body: { contentType: 'html', content: message.html },
        toRecipients: [{ emailAddress: { address: message.to } }]
      },
      saveToSentItems: false
    })
  }
}
```

Required env vars: `MS_GRAPH_TENANT_ID`, `MS_GRAPH_CLIENT_ID`, `MS_GRAPH_CLIENT_SECRET`,
`MS_GRAPH_SENDER_EMAIL` (e.g., `noreply@plaground.com.au`). Add to `.env.example` if missing.

**C. Gmail API adapter (apps/worker/src/lib/email/gmail-adapter.ts)**

Uses `googleapis` package with service account credentials.

```typescript
import { google } from 'googleapis'

export class GmailAdapter implements EmailProvider {
  private gmail

  constructor() {
    const auth = new google.auth.GoogleAuth({
      credentials: JSON.parse(process.env.GOOGLE_SERVICE_ACCOUNT_JSON!),
      scopes: ['https://www.googleapis.com/auth/gmail.send'],
      subject: process.env.GMAIL_SENDER_EMAIL  // domain-wide delegation: impersonate sender
    })
    this.gmail = google.gmail({ version: 'v1', auth })
  }

  async sendMail(message: EmailMessage): Promise<void> {
    // Build RFC 2822 message
    const raw = Buffer.from(
      `From: ${process.env.GMAIL_SENDER_EMAIL}\r\n` +
      `To: ${message.to}\r\n` +
      `Subject: ${message.subject}\r\n` +
      `Content-Type: text/html; charset=UTF-8\r\n\r\n` +
      message.html
    ).toString('base64url')

    await this.gmail.users.messages.send({
      userId: 'me',
      requestBody: { raw }
    })
  }
}
```

Required env vars: `GOOGLE_SERVICE_ACCOUNT_JSON` (full JSON string), `GMAIL_SENDER_EMAIL`.
Add to `.env.example` with placeholder values.

**A. Notify worker (apps/worker/src/processors/notify.ts)**

BullMQ Worker consuming `'notify'` queue, concurrency 10. Job names define the notification type:

Supported job names and their handling:

`'order-status-change'` — `{ orderId, newStatus, customerId? }`
1. Fetch order + customer email from DB.
2. Render `orderStatusChangeEmail({ customerName, orderId, newStatus, orderUrl })`.
3. Send via `emailProvider.sendMail`.
4. If customer has a linked Discord account (`user.discordAccount` not null): send DM via Discord.
5. Create `Notification` record: `{ tenantId, userId, channel: 'EMAIL', eventType: 'ORDER_STATUS_CHANGE', refId: orderId, sentAt: new Date() }`.

`'quote-ready'` — `{ quoteId, userId, estimateType }`
1. Fetch quote + user from DB.
2. Render `quoteReadyEmail({ customerName, quoteId, estimateType, totalAud, quoteUrl })`.
3. Send email + Discord DM if linked.
4. Create `Notification` record.

`'quote-decision'` — `{ quoteId, decision, customerId }`
1. Fetch quote + customer from DB.
2. Render `quoteDecisionEmail({ customerName, quoteId, decision, note, quoteUrl })`.
3. Send email + Discord DM if linked.
4. Create `Notification` record.

`'password-reset'` — `{ userId, resetUrl, expiresIn }`
1. Fetch user email from DB.
2. Render `passwordResetEmail({ resetUrl, expiresIn, storeName: 'PLAground' })`.
3. Send email only (no Discord for password resets — security boundary).

`'new-order-admin'` — `{ orderId, totalCents, customerEmail }` — for admins with Discord DM
configured. Fetches all admin users where `discordNotificationsEnabled = true`, sends Discord DM
to each.

**Idempotency:** Before processing any job, check if a `Notification` record already exists for
`{ userId, eventType, refId, channel }`. If yes, skip and return immediately (mark job complete).
This prevents duplicate delivery on BullMQ retries.

**Error handling:** Wrap the main job body in try/catch. On email provider error:
- Log error with `pino` structured log: `{ level: 'error', msg: 'email_send_failed', jobId, jobName, error: err.message }`
- Re-throw to allow BullMQ retry (up to 3 attempts with exponential backoff, 5s base delay).
- On exhaustion (DLQ): create `Notification` with `{ status: 'FAILED', failureReason: err.message }`.

**Worker configuration:**
```typescript
new Worker('notify', handler, {
  connection: redis,
  concurrency: 10,
  limiter: { max: 10, duration: 1000 },  // rate limit: 10 emails/second
})
```

**B. Discord integration (apps/worker/src/lib/discord/index.ts)**

Uses `discord.js` v14 with a bot client (not OAuth user tokens — bot has `Guilds` + `DirectMessages`
intents only). Bot is initialized once at worker startup.

```typescript
import { Client, GatewayIntentBits } from 'discord.js'

const discordClient = new Client({
  intents: [GatewayIntentBits.DirectMessages]
})
discordClient.login(process.env.DISCORD_BOT_TOKEN)

export async function sendDiscordDm(params: {
  discordUserId: string, content: string
}): Promise<void> {
  const user = await discordClient.users.fetch(params.discordUserId)
  const dm = await user.createDM()
  await dm.send(params.content)
}

export async function sendDiscordChannelMessage(params: {
  channelId: string, content: string
}): Promise<void> {
  const channel = await discordClient.channels.fetch(params.channelId)
  if (!channel || !channel.isTextBased()) throw new Error('Not a text channel')
  await channel.send(params.content)
}
```

**C. Discord OAuth + account linking API (apps/api/src/routes/v1/discord/index.ts)**

Two routes under `/api/v1/discord`:

`GET /api/v1/discord/connect` — authenticated customer (`requireAuth`). Redirects to Discord OAuth2
authorization URL:
`https://discord.com/api/oauth2/authorize?client_id={DISCORD_CLIENT_ID}&redirect_uri={encoded_callback}&scope=identify&response_type=code&state={csrf_state}`

CSRF state: store a short-lived token (UUID) in the user's session before redirect. Verify on
callback return.

`GET /api/v1/discord/callback` — authenticated customer. Query params: `code`, `state`.
1. Verify `state` matches session value (CSRF protection).
2. Exchange `code` for Discord access token via `POST https://discord.com/api/oauth2/token`.
3. Fetch Discord user: `GET https://discord.com/api/users/@me` with the access token.
4. Upsert `discordAccount` on the user: `{ discordUserId: user.id, discordUsername: user.username, discordAvatarUrl }`.
5. Return redirect to `/portal` with success message.

Required env vars: `DISCORD_CLIENT_ID`, `DISCORD_CLIENT_SECRET`, `DISCORD_REDIRECT_URI`. Add to
`.env.example`.

`DELETE /api/v1/discord/disconnect` — authenticated customer. Removes discord account link from user.
Sets `discordAccount` to null. Does NOT revoke the Discord token (no API endpoint needed).

`PUT /api/v1/admin/discord-notifications` — authenticated admin. Body: `{ enabled: boolean, discordUserId?: string }`. Sets `adminUser.discordNotificationsEnabled = enabled` and `adminUser.discordUserId = discordUserId`. Used by admin to opt into Discord DMs for new orders and print queue events.
